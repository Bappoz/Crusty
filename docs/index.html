Visão Geral do Rippler

Uma jornada pelo processo de compilação de C, desde o código-fonte até o código de máquina.

O Rippler é um compilador de C desenvolvido como projeto acadêmico na disciplina de Compiladores. Seu objetivo é transformar código-fonte C em código executável passando por todas as fases clássicas de um compilador moderno.

Este site documenta a arquitetura, as decisões de projeto e o funcionamento interno de cada fase. Cada capítulo corresponde a uma etapa do pipeline de compilação. A única fase concluída até o momento é a Análise Léxica.

O Pipeline de Compilação

O processo de compilação no Rippler é dividido em cinco fases sequenciais. A saída de cada fase alimenta diretamente a próxima:

01 Análise Léxica Concluída 02 Análise Sintática Em desenvolvimento 03 Análise Semântica Planejada 04 Repr. Intermediária Planejada 05 Geração de Código Planejada

Estrutura do Projeto

O código-fonte está organizado em módulos que espelham as fases do compilador. Todos os módulos compartilham utilitários comuns via src/common/.

Estrutura de diretórios 
src/
├── lexer/          # Análise Léxica (scanner, tokens, regras)
│   ├── scanner.rs  # Loop principal de varredura
│   ├── tokens/     # TokenKind, Token, ByteSpan
│   └── rules/      # Regras de literais, operadores, identificadores
│
├── parser/         # Análise Sintática (gramática, AST)
├── analyser/       # Análise Semântica (tipos, escopo)
├── codegen/        # Geração de Código
│
└── common/         # Utilitários compartilhados
    ├── errors/     # Sistema de diagnósticos e relatórios
    ├── input/      # SourceFile, ByteSpan
    └── utils/      # char_utils e helpers

Tecnologias

Componente Tecnologia Justificativa
linguagem Rust Segurança de memória, performance e sistema de tipos expressivo
leitura de arquivo memmap2 Memory-mapped I/O para leitura eficiente de arquivos grandes
testes Rust built-in Testes unitários e de integração com #[test]

Fluxo de Dados

Um arquivo .c é lido pelo SourceFile via memory mapping. O Scanner consome o source char a char produzindo uma lista de Tokens. Erros léxicos são coletados em diagnostics sem interromper a varredura.

Recuperação de erros

O Rippler adota a estratégia de error recovery em cada fase: ao encontrar um erro, o compilador registra o diagnóstico e continua processando o restante do código. Isso permite que múltiplos erros sejam reportados em uma única execução, melhorando a experiência do desenvolvedor.
